Opening
Are you using Revit materials?

"Virtual show of hands — who's actively using Revit materials on projects right now? And who's been wanting to but hasn't gotten there yet?"

Either way, materials are worth the investment. Here's why:

  • Appearance is tied to the model. Assign a material once and it shows up correctly in every view, every rendering, every export — no manual re-applying per view or visual style.
  • Data lives in the model. Materials carry embedded parameters — manufacturer info, product codes, specs — that can be scheduled and reported directly from Revit. Great for finish schedules, material takeoffs, and documentation packages.
  • Fast, iterative design. Swap a material globally across a model in seconds. Ideal for quick client options, design reviews, or pulling together a fast rendering without rebuilding anything.
The Problem
Why materials have been such a pain

"Has anyone ever opened a model and half the materials are just... gray? Or a texture that worked perfectly on your machine looks broken for everyone else?"

That's the thing we fixed.

The root cause is simple: Revit stores image maps as file paths. If that path doesn't exist on your machine — saved to someone's desktop, or a server you don't have access to — the material breaks. Every time. For everyone.

And the Revit Material Browser itself isn't much help. Dense, hard to navigate, and not exactly inspiring when you're trying to pick finishes.

Setup
Steps

One toggle gets everything running. Settings → Model Tracking → Materials. Admin only.

1
Admin enables Materials in Settings → Model Tracking.
2
Open your Revit template or a project with your standard material set.
3
In the add-in: Add to Library → Materials — select all or choose specific ones.
4
Done. Those materials are in the library, image maps handled, available to the whole team.
Quick Start
Let the model do the work

If your team is already tracking models, Kinship can build your material library for you.

1
Track your models. Enable model tracking in Settings — Kinship will start picking up materials automatically as models sync.
2
Check Most Used. Materials that appear frequently across tracked models surface here — a ready-made shortlist of what your team actually uses.
3
Populate your library. Add the most-used materials from Most Used in one go. Your library starts with real project content, not a blank slate.
Notes
Good to know
Good to know: Turning this on doesn't populate your library automatically. Existing content is untouched — materials only appear in the library when you explicitly add them.
Using an .adsklib file? Load the materials into a Revit model first, then add to library from there. One extra step, but just the once.
The Solution
What Kinship now does

Kinship gives your materials a proper home. Visual, searchable, and reliable for the whole team.

  • Image map headaches → gone. Once a material is in Kinship, image file locations are handled automatically for every user. It just works.
  • A library that's actually nice to use. Large realistic thumbnails, organized by class, searchable by keyword. A material browser you'd actually want to open.
  • Visibility into your projects. Tracked models show which materials are in use — making it easier to keep things consistent across the team.
Web App
Library → Materials section
  • Browse by Class — the recommended way to navigate. Masonry, Glass, Wood, etc. Custom classes can be created too.
  • Browse by Appearance Asset Type — 19 fixed Revit types (Opaque, PBR, etc.). Useful for rendering-focused workflows.
  • All Materials — full list, sortable and filterable by class, asset type, list, or Revit version.
  • Utilities — recently added, most used, unused, recently edited.
Web App
The material item page
  • Large realistic thumbnail with consistent colors and hidden line previews — not Revit's sad gray swatch.
  • Identity info — class, asset type, Revit version.
  • Graphics — surface pattern, cut pattern, and shading previews.
  • Appearance — image map previews and asset info.
  • Description — admin-editable, searchable in both the web app and the add-in.
  • Manufacturer links & Documentation — two flexible URL fields for spec sheets, product pages, or any external links. These are Kinship-side fields (like Description) — separate from the Manufacturer parameter in Identity. Use them however works for your team.
  • Comments — leave feedback or flag questions on any material.
  • Similar materials — matched by class and asset type.
  • Files — attach PDFs, images, or any supporting docs.
Add-In
Searching directly from Revit
  • Materials appear in search alongside families — filter to materials to narrow down.
  • Preview before loading — click any material to see description and details first.
  • Load directly into your open project with one click.
  • Add to Library from any open model or family via Add to Library → Materials. You can also add materials directly from tracked models — a great way to build your library from real project content.
